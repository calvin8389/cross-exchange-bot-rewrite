# 异常场景处置手册

## 启动后发现历史残仓
1. 停止自动开新仓。
2. 对照数据库 `positions` 与交易所真实持仓。
3. 双边都在则恢复为 `HOLDING`；仅单边在则进入 `ERROR`，先人工降风险。

## 一腿成交、另一腿失败
1. 立即执行同 symbol 回滚。
2. 回滚成功则记录 `OPENING_ROLLBACK` 并回到空仓态。
3. 回滚失败则记录 `OPENING_ROLLBACK_FAILED`，输出残仓信息并人工接管。

## 持仓中出现单边残仓
1. 记录 `UNHEDGED_POSITION`。
2. 将状态切换到 `ERROR`，禁止继续扫描/补仓。
3. 人工确认方向、数量、成交价后优先消除裸露风险。

## 平仓重试失败
1. 记录 `CLOSING_FAILED`，包含残仓交易所、方向、数量、浮盈亏。
2. 状态进入 `ERROR`。
3. 人工手平后再修复数据库与复盘记录。

## API / funding 拉取持续异常
1. 连续失败达到阈值后查看 `POSITION_QUERY_FAILED`、`FUNDING_REFRESH_FAILED`、`FUNDING_HISTORY_FAILED` 事件。
2. 暂停新增风险；必要时改为人工盯盘并平仓。
