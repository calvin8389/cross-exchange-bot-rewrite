# 资金费率套利实盘风控清单

## 开机前检查
- 校验 `ACTIVE_EXCHANGES`、账户凭证与目标环境一致。
- 确认各交易所余额、可用保证金、持仓模式可读。
- 确认 `bot_config.json` 中监控币种、APR 阈值、最大价差、持仓时长、并发仓位、暴露上限符合当日策略。
- 确认数据库可写、日志可写、`close_all.sh` 存在且可执行。
- 启动后先执行只读健康检查，再进入恢复或扫描流程。

## 扫描前风控
- 仅扫描 `symbols_to_monitor` 白名单。
- 候选机会必须同时满足价差上限与扣除手续费/滑点/冲击成本后的净 APR 阈值。
- 任一交易所元数据缺失、盘口异常、资金费率刷新失败时不允许开仓。

## 下单前风控
- 双腿都必须校验 `tick`、`step`、`min_order_size`、`min_notional`。
- 双腿都必须校验可用保证金足够覆盖目标名义金额。
- 开仓前校验单币、单所、组合总暴露与组合回撤限制。
- 下单前写入结构化事件，记录 pair、价格、数量、名义金额、风险敞口快照。

## 持仓与平仓风控
- 持仓轮询持续检查双腿是否对称；出现单腿残仓立即进入 `ERROR`。
- 资金费率、持仓查询、funding history 连续失败达到阈值后触发错误事件。
- 触发 APR 跌破、止损、最大持仓时长时，平仓必须写明 `close_reason`。
- 平仓重试耗尽后必须记录结构化残仓信息并升级 `CLOSING_FAILED` / `ERROR_STATE`。
